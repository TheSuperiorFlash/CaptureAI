# CaptureAI Database Guide

> **Self-update rule:** When you add/change tables, columns, indexes, or views — update this file. When you add a migration, also update [api/migrations/README.md](migrations/README.md).

Cloudflare D1 (SQLite) database. Schema defined in `api/schema.sql`, single migration in `api/migrations/`.

## Quick Access

1. [Cloudflare Dashboard](https://dash.cloudflare.com/) -> **Workers & Pages** -> **D1** -> **captureai-db** -> **Console**
2. Or use CLI: `wrangler d1 execute captureai-db --command "YOUR SQL"`

## Schema

### users

| Column | Type | Notes |
|--------|------|-------|
| id | TEXT PK | UUID generated by application |
| email | TEXT NOT NULL | User email |
| license_key | TEXT UNIQUE NOT NULL | Format: XXXX-XXXX-XXXX-XXXX-XXXX |
| tier | TEXT | `basic` or `pro` |
| subscription_status | TEXT | `active`, `inactive`, `cancelled`, `past_due` |
| stripe_customer_id | TEXT UNIQUE | Stripe customer reference |
| stripe_subscription_id | TEXT | Stripe subscription reference |
| created_at | TEXT NOT NULL | ISO timestamp |
| updated_at | TEXT NOT NULL | ISO timestamp; updated on every mutation |

**Indexes:** email, stripe_subscription_id, tier, subscription_status (all explicit); license_key, stripe_customer_id (implicit via UNIQUE)

### usage_records

Per-request analytics. Writes are best-effort (logged but non-blocking). Rows older than 90 days are purged by the daily cron.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER PK | Auto-increment |
| email | TEXT NOT NULL | User email |
| prompt_type | TEXT | `answer`, `ask`, `auto_solve`, etc. |
| model | TEXT | `low`, `medium`, `high` |
| input_tokens, output_tokens | INTEGER | Token counts |
| total_cost | REAL | Cost in USD |
| cached | INTEGER | `1` = cache hit, `0` = miss |
| response_time | INTEGER | Milliseconds |
| created_at | TEXT NOT NULL | ISO timestamp |

**Indexes:** `idx_usage_records_email_date` (email, created_at)

### usage_daily

Authoritative daily aggregates. Primary source for rate-limit checks (O(1) composite PK point-lookup). The composite PK `(email, date)` creates its own index.

| Column | Type | Notes |
|--------|------|-------|
| email | TEXT | Composite PK with date |
| date | TEXT | YYYY-MM-DD |
| request_count | INTEGER | Total daily requests |
| cached_request_count | INTEGER | Requests that were cache hits |
| input_tokens, output_tokens | INTEGER | Daily token totals |
| total_cost | REAL | Daily cost |

### webhook_events

Stripe event deduplication. Rows older than 24 hours are pruned after each webhook delivery and by the daily cron.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER PK | Auto-increment |
| event_id | TEXT UNIQUE NOT NULL | Stripe event ID |
| event_type | TEXT NOT NULL | Stripe event type string |
| processed_at | TEXT NOT NULL | Set to datetime('now') on insert |
| webhook_timestamp | TEXT NOT NULL | Timestamp from Stripe event payload |

### verification_codes

Email OTP codes for login and tier-switch. Cleaned up daily by cron.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER PK | Auto-increment |
| email | TEXT NOT NULL | User email |
| code | TEXT NOT NULL | 6-digit OTP |
| action | TEXT NOT NULL | `login` or `tier_switch` |
| tier | TEXT | Target tier (NULL for login OTPs) |
| created_at | TEXT NOT NULL | datetime('now') |
| expires_at | TEXT NOT NULL | 10-minute TTL |
| used | INTEGER NOT NULL | `1` = consumed, `0` = active |

**Indexes:** `idx_verification_codes_lookup` (email, action, used, expires_at) — covers the hot lookup path

### subscription_events

Immutable audit log. One row per subscription state change. Never updated or deleted. Use this to answer billing disputes.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER PK | Auto-increment |
| email | TEXT NOT NULL | User email |
| event_type | TEXT NOT NULL | `signup`, `checkout_completed`, `tier_changed`, `cancelled`, `payment_succeeded`, `payment_failed`, `subscription_updated`, `subscription_lapsed` |
| from_tier | TEXT | Tier before change |
| to_tier | TEXT | Tier after change |
| from_status | TEXT | Status before change |
| to_status | TEXT | Status after change |
| stripe_event_id | TEXT | Stripe event or object ID |
| metadata | TEXT | JSON blob for amounts, proration, etc. |
| created_at | TEXT NOT NULL | ISO timestamp |

**Indexes:** `idx_subscription_events_email` (email, created_at)

### Views

- **total_usage** — Grand totals plus per-prompt_type and per-model breakdowns (reads usage_records)
- **user_usage** — Per-user aggregate statistics (reads usage_records)
- **total_usage_daily** — All-time totals including cache hit count (reads usage_daily; authoritative)

## Common Queries

```bash
# All users
wrangler d1 execute captureai-db --command "SELECT * FROM users;"

# Daily usage summary
wrangler d1 execute captureai-db --command "
  SELECT date, SUM(request_count) as requests, SUM(cached_request_count) as cached, SUM(total_cost) as cost
  FROM usage_daily GROUP BY date ORDER BY date DESC LIMIT 7;"

# Cost by model
wrangler d1 execute captureai-db --command "
  SELECT model, COUNT(*) as requests, SUM(total_cost) as total_cost
  FROM usage_records GROUP BY model;"

# Active Pro subscriptions
wrangler d1 execute captureai-db --command "
  SELECT email, subscription_status, created_at
  FROM users WHERE tier = 'pro' AND subscription_status = 'active';"

# Subscription history for a user
wrangler d1 execute captureai-db --command "
  SELECT event_type, from_tier, to_tier, from_status, to_status, created_at
  FROM subscription_events WHERE email = 'user@example.com' ORDER BY created_at;"
```

## Maintenance

```bash
# Backup
wrangler d1 export captureai-db --output=./backup-$(date +%Y%m%d).sql

# Run initial schema (fresh database)
wrangler d1 execute captureai-db --file=./migrations/0001_initial_schema.sql

# Manual cleanup (also runs automatically via daily cron)
wrangler d1 execute captureai-db --command "DELETE FROM usage_records WHERE created_at < datetime('now', '-90 days');"
wrangler d1 execute captureai-db --command "DELETE FROM webhook_events WHERE processed_at < datetime('now', '-24 hours');"
wrangler d1 execute captureai-db --command "DELETE FROM verification_codes WHERE expires_at < datetime('now') OR used = 1;"
```
